# Changesets

Hello and welcome! This folder has been automatically generated by `@changesets/cli`, a build tool that works
with multi-package repos, or single-package repos to help you version and publish your code. You can
find the full documentation for it [in the changesets repository](https://github.com/changesets/changesets).

## Adding a changeset

To add a changeset, run `pnpm changeset` in the root of the repo. It will ask you:

1. What kind of change (major/minor/patch)
2. A summary of what changed

This creates a markdown file in this `.changeset/` directory that describes the change.

## Example changeset file

```markdown
---
"groot": patch
---

fix(core): handle null values in Boom error handler
```

## Releasing

When ready to release, run:

1. `pnpm changeset version` — consumes changeset files, bumps version, updates CHANGELOG.md
2. Commit and push — the CI will create a GitHub Release

Or simply merge to `main` and the release workflow handles it automatically.
